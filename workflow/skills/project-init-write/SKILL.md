---
name: project-init-write
description: Reads .claude/project-init-session.md written by /project-init-new, shows a summary, requires CONFIRM, then writes all project config files, docs, CLAUDE.md, and spec-000.
---

Read the project-init session file and write all project outputs after explicit confirmation.

⛔ No project files written before CONFIRM.

---

## Step 1 — Read and validate session file

Read `.claude/project-init-session.md`. If the file does not exist, print `❌ No session file found at .claude/project-init-session.md — run /project-init-new first.` and stop.

If the file has `_Status: complete_`, warn that it has already been applied and re-running will overwrite project files. Ask `Continue anyway? (yes / no)` ⛔ STOP. If "no" — halt.

Validate required fields are non-empty: `name`, `backend_lang`, the `## Architecture` section, and the `## Spec-000` section. If any are missing, list them and stop with: `❌ Session file is incomplete — fill the missing fields, then re-run.`

---

## Step 2 — Show summary

Count the entries in the `## Decisions` section. Present:

```
Ready to write — please confirm.
──────────────────────────────────────────────────
Project:      [name]
Stack:        [backend_lang] / [frontend_lang] / [db_type]
Orchestrator: [orchestrator]
Patterns:     [patterns_include]
Decisions:    [N] to record
First spec:   spec-000-[name]

Files that will be written:
  .claude/workflow-config.md
  .claude/workflow-infra.md
  .claude/workflow-git.md
  .claude/workflow-smoke.md
  docs/ARCHITECTURE.md
  docs/DECISIONS.md
  docs/MAP.md                  ← auto-generated component index (regenerated by /decision-sync)
  docs/PATTERNS.md             ← project's selected-patterns list (generated from patterns_include)
  CLAUDE.md                    ← project description + ## Project Rules appended
  specs/spec-000-[name].md
  [if design extracted]
  docs/DESIGN-SYSTEM.md
  docs/COMPONENT-LIBRARY.md
  .rossflow/design-source-index.json
  .rossflow/design-tokens.json
  specs/spec-001-design-tokens-and-primitives.md
  specs/spec-002-composite-components.md
──────────────────────────────────────────────────
Type CONFIRM to write all files, or tell me what to change.
```

⛔ STOP. Wait for CONFIRM or explicit equivalent ("yes", "do it", "write it"). If developer requests changes: update `.claude/project-init-session.md` accordingly, re-show the summary, do not write project files until CONFIRM.

---

## Step 3 — Write outputs (only after CONFIRM)

Write in this order. Print each filename as it is written.

**Re-run guard:** if a target file already has content beyond template placeholders, show its first line of existing content alongside the new value and ask `[filename] already has content. Overwrite? (yes / skip)`. ⛔ STOP per file. Wait for response before writing.

### 1. `.claude/workflow-config.md`
Populate from session `## Stack` and `## Patterns` sections. Field names and order match `workflow/config-templates/workflow-config.md` exactly. Fixed values: `specs_dir`=specs, `smoke_tests_dir`=smoke-tests, `bugs_dir`=bugs, `docs_dir`=docs. Preserve template comments and section headers. Empty session values stay as blank placeholders.

### 2. `.claude/workflow-infra.md`
Populate from session `## Infra` section. Preserve template comments and headers.

### 3. `.claude/workflow-git.md`
Populate from session `## Git` section. Preserve template comments and headers.

### 4. `.claude/workflow-smoke.md`
Populate from session `## Smoke` section. Preserve template comments and headers.

**Stack defaults** (apply if session value is missing or default; override only if a Phase 6 DEC said otherwise):
- React Native + Expo → `ui_driver`=maestro, `test_runner`=jest. Disabling unit tests REQUIRES a DEC citing rationale.
- Web React (Next.js) → `ui_driver`=playwright, `test_runner`=vitest.

### 5. `docs/ARCHITECTURE.md`
Write session `## Architecture` content VERBATIM (do not summarize). Use `workflow/docs-templates/ARCHITECTURE.md` 13-section structure as the literal scaffold.

### 6. `docs/DECISIONS.md`
Use the ADR schema from `workflow/docs-templates/DECISIONS.md`. Each DEC = full ADR block (Status / Context / Decision / Consequences / Verifies | Unverifiable), not a table row. Use today's date for any entry without a date.

### 7. `docs/PATTERNS.md` (generated, no template copy)
Generate fresh from `patterns_include`. Read one-line summaries from `docs/patterns/INDEX.md`. Write:

```markdown
# Patterns

> Project-selected coding conventions. For the rule body of any item below, read `docs/patterns/<stem>.md`.
> Framework index with all available patterns: `docs/patterns/INDEX.md`.
> Skills grep THIS file by stack keyword and read only the matching pattern files — never bulk-load.

## Always-on
- ✅ principles — <copied description from INDEX.md>
- ✅ arch — <copied description from INDEX.md>

## Selected for this project
<!-- one bullet per stem in patterns_include except principles, arch, in INDEX.md order -->
- ✅ <stem> — <copied description from INDEX.md>

## Not applied
<!-- one bullet per framework pattern not in patterns_include and not always-on -->
- ⬜ <stem> — not applicable (<short reason tied to stack, e.g. "no Django in stack")
```

Do not include the framework `INDEX.md` itself in either list — it lives at `docs/patterns/INDEX.md` separately.

### 8. `CLAUDE.md`
Reference `.claude/workflow-instructions.md` (existing pattern — preserve). Append after existing content (do not modify what is already there):

```

[project description from session ## Project → description field]

## Project Rules
[rules from session ## Project rules section, one bullet per line]
```

Do NOT inline framework-global rules; those live in `.claude/workflow-instructions.md`.

### 9. `specs/spec-000-[name].md`
Write session `## Spec-000` content verbatim. Section order and names must match `specs/spec-template.md` — no custom sections, no omitted sections.

### 10. `docs/MAP.md`
Use the schema from `workflow/docs-templates/MAP.md` (the GENERATED banner + `| Component | Path | README |` header). Table starts empty — `decision-sync` populates and regenerates it from per-component READMEs. No `docs/ctx/` folder, no legacy ctx columns. Do not hand-edit after generation.

### 11. Design source outputs (only if `design_source_status == "extracted"`)
- Copy `design-source-index.json` and `design-tokens.json` to `<target_project>/.rossflow/`.
- Write `<target_project>/docs/DESIGN-SYSTEM.md` from extraction output (template: `workflow/docs-templates/DESIGN-SYSTEM.md`).
- Write `<target_project>/docs/COMPONENT-LIBRARY.md` from extraction output (template: `workflow/docs-templates/COMPONENT-LIBRARY.md`).

Skip this section if `design_source_status != "extracted"`.

### 12. Spec ordering after spec-000
If `design_source_status == "extracted"`:
- Generate `specs/spec-001-design-tokens-and-primitives.md` stub (inputs: DESIGN-SYSTEM.md).
- Generate `specs/spec-002-composite-components.md` stub (inputs: COMPONENT-LIBRARY.md).

spec-003+ feature/screen specs come later via `/spec-create`.

---

## Step 4 — Mark session complete

Update `.claude/project-init-session.md`: change `_Status: awaiting-confirm_` to `_Status: complete_`.

---

## Step 5 — Bootstrap tools

Print:

```
── tools-bootstrap ────────────────────────────────────
```

Invoke `/tools-bootstrap`. This checks and installs all tools required by the project's tech stack. The skill will prompt before installing anything.

After `/tools-bootstrap` completes, print:

```
✅ project-init-write complete.

Next steps:
1. Review docs/ARCHITECTURE.md and docs/DECISIONS.md — adjust if needed
2. Run: /spec-implement spec-000-[name]
3. After implementation: /spec-smoke-test spec-000-[name]
4. Then: /ship
```
