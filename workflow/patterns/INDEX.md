# Patterns Library — Framework Index

> Routing index for the rossflow patterns library. Skills load patterns by stack at project init; agents grep by keyword at spec time.
> The project's own selected-patterns list is `docs/PATTERNS.md` (generated by `project-init-write`). Do not confuse the two.

---

## Always Load

These apply to every implementation task regardless of stack.

- [principles.md](principles.md) — Language-agnostic coding rules: function size, naming, early return, pure functions, DRY, YAGNI, comments, single responsibility, cyclomatic complexity, hard caps.
- [arch.md](arch.md) — System architecture: three-layer model, HTTP/DB layer boundaries, module interfaces, ports and adapters, god object detection, unidirectional data flow, error handling at boundaries.

---

## Load When Relevant

Selected during `project-init-new` Phase 4 based on the approved stack. The chosen file stems are stored as `patterns_include` in `.claude/workflow-config.md`.

### Frontend

- [typescript.md](typescript.md) — TypeScript: strict mode, no `any`, discriminated unions, `as const` tokens, closure-stale guards, branded types.
- [reactnative.md](reactnative.md) — React Native: StyleSheet.create, typed navigation params, Platform.select, centralised API client, useEffect cleanup, native module error handling, FlatList.
- [nextjs.md](nextjs.md) — Next.js App Router: Server vs Client components, server-side data fetching, typed API client, dynamic imports, error.tsx, loading.tsx.

### State management (React/RN)

Selection rule: always include `state-management` when frontend has React/RN, plus exactly one library file based on the project's choice.

- [state-management.md](state-management.md) — Library-agnostic principles: when to lift state, when to reach for global, immutability, atomic mutations, derived state, memoization.
- [state-redux-toolkit.md](state-redux-toolkit.md) — Redux Toolkit: slices, RTK Query, store setup, typed hooks, selectors.
- [state-zustand.md](state-zustand.md) — Zustand: typed stores, slicing, persist middleware, selector subscriptions.

### Persistence (client-side)

- [client-persistence.md](client-persistence.md) — Client-side KV/secure/structured storage: typed wrapper, versioned keys, atomic writes, hydration loading, migration. Adapter examples: AsyncStorage, MMKV, SecureStore, SQLite.

### Backend

- [fastapi.md](fastapi.md) — FastAPI: route definition, Pydantic request validation, Depends() injection, HTTPException error handling, BackgroundTasks, structured logging, router organisation.
- [django.md](django.md) — Django REST Framework: model conventions, serializer fields, ViewSet vs APIView, permissions, settings structure, signal usage, QuerySet manager methods.
- [sqlalchemy.md](sqlalchemy.md) — SQLAlchemy async: async session factory, Mapped[T] columns, select() queries, transaction scope, migration-only schema changes, explicit relationship loading, session lifecycle.

### Testing (Python)

- [pytest.md](pytest.md) — Pytest async: fixture scope, asyncio_mode=auto, parametrize, monkeypatch/AsyncMock, AsyncClient+ASGITransport, conftest structure, assertion messages, test naming.

## Reading rule

Read `principles` + `arch` always. For everything else, grep this index by stack keywords. Read only matched files. Never bulk-read all patterns.

## File budget

≤200 lines per file. Split if larger.
